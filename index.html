关于考研
2020年05月21日 1.2k 字 大概 4 分钟

目前已经历两次考研，第一次未过国家线，第二次过国家线未过院校线，为了更好发展，在此总结。

第一次考研(2019)

一志愿-浙江理工大学-专业方向-工程心理学-政治一-英语一-心理学综合(312)

个人原因

  • 个人准备不够充分，加上医科院校为期五个月的较高强度实习，因此一战我更多是尝试性质。
  • 临考马虎大意，英语阅读十分未涂，剖析内心后我认为是因为我对专业认同存疑，由潜意识上升到意识层面，表现为临考失误。

院校因素

  • 浙理是国内除浙大之外最好的工业心理学院校，在葛列众老师一时兴起（搓麻将）的带领下，实力强劲，虽然其已被浙大返聘，但余音绕梁。

时代因素

  • 由于是心理学方向的HCI，许多人也像我一样想追赶大数据时代的潮流。
  • 社会日益内卷，中美贸易摩擦加剧就业危机，使考研人数与日俱增。

其他因素

  • 不限制本专业，加上市面上众多的心理学考研机构，信息严重不对称，本专业毫无优势。

第二次考研(2020)

一志愿-华中师范大学-专业方向-网络行为与工程心理学-政治一-英语一-心理学综合(312)

个人原因

  • 个人准备仍不够充分，二战选择在家备考，在家始终处于低效率状态。
  • 对家人的认知不够理性，家里人虽然表面支持，但会无意识地施加干扰，严重影响备考心态。
  • 毕业后未能妥善处理情绪，那几日以泪洗面（真），大概流光了这辈子的泪水，虽然原因不明。

院校因素

  • 华中师范2020新开设了网络行为与工程心理学方向，并且于十月加入了312统考，我认为是个机会(×)。
    事实告诉我认知不充分就要付出代价，华中师范往年都是不分方向统一划线，今年也是如此，并且211直属师范加入统考，引来了大量的机构考生，生源质量堪称2020第一档。

时代因素

  • 由于是心理学方向为数不多与大数据时代挂钩的专业方向，并且学院与腾讯开展了合作意向(我报考原因之一)，竞争必然激烈。
  • 新冠疫情使得就业危机加剧，大部分地区心理学扩招，可能是出于防控角度，武汉地区心理学没有扩招名额。

    其他原因

  • 不限制本专业，加入市面上众多的心理学考研机构，信息严重不对称，本专业毫无优势。

总结反思

院校选择

  • 尽量避开心理学考研机构推荐的院校，因为心理学考研机构推荐的院校往往竞争激烈，有人笑谈进了院校群发现都是一个机构的人。
  • 如果想从事学术，尽量根据选择具有脑与认知神经科学实验室的院校，因为认知神经科学论文好发Sci，如：深圳大学、陕西师范大学、中科院、浙江大学、四川师范大学脑所、南方医科大学（可出国）。
  • 如果想从事工业，推荐院校有：浙江理工（工程心理学or用户体验）、天津师范(用户体验)、中科院（工业心理学实验室）、浙江大学(工业心理学重点实验室)。
  • 如果想从事临床，建议选择医学院，推荐院校有：北大医学部、中南大学二院（保护本专业）、等等。

    专业方向

  • 由于我对心理学发展存疑，并且我发现备考心理学研究生所学知识与我所想从事方向差别甚大，我决定放弃心理学之路。
  • 我个人奉行实用主义，实在无法接受备考用书与研究生方向关联甚微，对工业心理学外的任何心理学方向我都没有兴趣，不想耽误老师，更不想耽误自己，在此对魏老师说声抱歉。
  • 我问了自己这样一个问题：
    如果不给我收入，我会去做什么？
    我最终想到是游戏开发，因此看到中国传媒大学的人机交互与游戏开放方向的我，决定放弃调剂，再博一把。

愿他年某月，再看此时，无悔今朝。

如何通过Hexo搭建个人博客（Windows+Git+Node.js+Hexo）
2020年05月20日 935 字 大概 4 分钟

之所以写这个教程，一方面是因为我在搭建博客过程中遇到了不少问题，便于以后回忆；另一方面是整合自己相关的知识形成体系，便于结构化。

环境配置

Node.js

什么是Node.js?

Node.js是采用ChromeV8运行代码，基于JavaScript编写，以实现服务器端编程的开放源代码，在这里负责博客的构建渲染。

为什么用Node？

Node.js配置

Git

什么是Git?

Git是分布式版本控制工具，在这里负责博客与服务器之间信息的上传和下载。

为什么用Git？

Git配置

Hexo

什么是Hexo？

Hexo是一种快速、简洁且高效的博客框架，基于Node.js引擎，生成静态页面。

为什么用Hexo？

  • 超快速度
  • 一键部署
  • 支持markdown语法
  • 插件与可扩展性

Hexo的配置

首先我们先检测一下Node和Git的版本，确保环境没有问题，打开Git-bash，键入：

$ node -v
$ git --version

如果你的环境没有问题，那么我们可以开始安装Hexo，键入：

$ npm install hexo-cli -g #安装hexo框架到环境根目录

安装完成后，新建一个文件夹作为你博客的根目录，在该文件夹右键Git-bash，键入:

$ hexo init blog #生成基于hexo的博客文件夹
$ cd blog           #打开该文件夹
$ npm install      #安装hexo框架
$ hexo server     #上传到本地的hexo服务器，即http://localhost:4000/

无意外情况会输出如下结果：

$ INFO  Start processing
$ INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

Hexo的常用指令

$ hexo generate #构建博客，可缩写为hexo g
$ hexo server #上传到本地服务器，地址为http://localhost:4000/，可缩写为hexo s
$ hexo deloyer #上传到网络服务器，地址一般为[username].github.io/[blogname].github.io，可缩写为hexo d

博客配置

打开博客根目录的_config.yml文件。

基本配置

找到_config.yml文件的以下关键词并根据自己需要修改。

#Sites
title: [博客主标题]
subtitle:[博客副标题]
description:[博客简介]
author: [博客所有人名]
language:[语言]（中国简体为zh-cn，英文为en）
timezone:[时区]

主题选择

博客的基本属性配置完毕，接下来要如何美化博客呢？
Hexo提供了scheme社区，许多大佬基于Hexo设计了自己的博客并上传到主题社区，每个主题的具体设置不同，详见作者的说明手册（如果未按照手册设置，很容易部署失败)，当然你也可以DIY。
根据作者的说明手册在博客根目录右键Git-bash，键入作者提供的主题源，这里以我使用的Nexmoe为例：

$git clone  https://github.com/MOxFIVE/hexo-theme-nexmoe.git themes/nexmoe #从主题源拷贝主题文件到themes/nexmoe目录下

再找到博客根目录的_config.yml文件的theme关键词并根据自己需要修改。
(nexmoe还要求拷贝nexmoe目录下的config.example.yml并命名为config.yml，理由详见nexmoe的说明手册)

#Extensions
##Plugins: https://hexo.io/plugins/
##Themes: https://hexo.io/themes/
theme: [你想要的主题]
