环境配置

Node.js（推荐长期支持版，不容易报错）

  • Node.js是采用ChromeV8运行代码，基于JavaScript编写，以实现服务器端编程的开放源代码，在这里负责博客的构建渲染。

Git

  • Git是分布式版本控制工具，往往用于软件代码的云端托管，在这里负责博客与服务器之间信息的上传和下载。

Hexo

什么是Hexo？

  • Hexo是一种快速、简洁且高效的博客框架，基于Node.js引擎，生成静态页面。

为什么用Hexo？

  • 超快速度
  • 一键部署
  • 支持markdown语法
  • 插件与可扩展性

Hexo的配置

  • 首先我们先检测一下Node和Git的版本，确保环境没有问题，打开Git-bash，键入：
    $ node -v
$ git --version
  • 如果你的环境没有问题，那么我们可以开始安装Hexo，键入：
    $ npm install hexo-cli -g #安装hexo框架到环境根目录
  • 安装完成后，新建一个文件夹作为你博客的根目录，在该文件夹右键Git-bash，键入:
    $ hexo init blog #生成基于hexo的博客文件夹
$ cd blog           #打开该文件夹
$ npm install      #安装hexo框架
$ hexo server     #上传到本地的hexo服务器，即http://localhost:4000/
  • 无意外情况会输出如下结果：
    $ INFO  Start processing
$ INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

Hexo的常用指令

$ hexo generate #构建博客，可缩写为hexo g
$ hexo server #上传到本地服务器，地址为http://localhost:4000/，可缩写为hexo s
$ hexo deloyer #上传到网络服务器，地址一般为[username].github.io/[reponame].github.io，可缩写为hexo d

博客配置

  • 打开博客根目录的_config.yml文件。

基本配置

  • 找到_config.yml文件的以下关键词并根据自己需要修改。
    #Sites
title: [博客主标题]
subtitle:[博客副标题]
description:[博客简介]
author: [博客所有人名]
language:[语言]（中国简体为zh-cn，英文为en）
timezone:[时区]bgii/-

主题选择

  • 博客的基本属性配置完毕，接下来要如何美化博客呢？
  • Hexo提供了scheme社区，许多大佬基于Hexo设计了自己的博客并上传到主题社区，每个主题的具体设置不同，详见作者的说明手册（如果未按照手册设置，很容易部署失败)，当然你也可以DIY。
  • 根据作者的说明手册在博客根目录右键Git-bash，键入作者提供的主题源，这里以我使用的Nexmoe为例：
    $git clone  https://github.com/MOxFIVE/hexo-theme-nexmoe.git themes/nexmoe #从主题源拷贝主题文件到themes/nexmoe目录下
  • 再找到博客根目录的_config.yml文件的theme关键词并根据自己需要修改。
    (nexmoe还要求拷贝nexmoe目录下的config.example.yml并命名为config.yml，理由详见nexmoe的说明手册)
    #Extensions
##Plugins: https://hexo.io/plugins/
##Themes: https://hexo.io/themes/
theme: [你想要的主题]

博客托管

  • 现在我们已经有了本地的博客，要如何让别人可以访问我们的博客呢？对，就是将我们的博客上传到互联网上，如果有条件可以通过自己的私人服务器实现，但更常见的是将博客数据上传到云端。

选择服务商并上传博客数据

  • 目前我已知国内提供类似服务的有Gitee、Coding，国外提供类似服务的有Github，我们这里选择Github。

部署配置

  • 找到_config.yml文件的以下关键词并根据自己需要修改，master是该仓库的主要分支。
    # Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
type: git
repo:
   github: git@github.com:[用户名]/[仓库名],master     #！！若为git@github.com:[用户名]/[用户名.github.io],master则会自动开启github pages，否则需要手动开启，路径为：`Your repositories`-`Settings`-`Github Pages`
   coding: git@e.coding.net:[用户名]/[仓库名].git,master     #coding pages在我使用的时候已经停止pages服务
   gitee: git@gitee.com:[用户名]/[仓库名],master       #gitee pages服务需要手动开启，路径为：`仓库`-`服务`-`Gitee Pages`

通过SSH免密上传本地数据到云端

  • 首先我们要登记自己博客的账号密码并生成对应的SSH密钥，在博客根目录右键Git-bash，根据你在服务商注册的信息键入：
    $ ssh-keygen -t rsa -b 4096 -C "[你的邮箱]" #将用户名和邮箱通过RSA算法进行加密
  • 之后会显示如下，如果想要密码就自己输入密码，如果不想要密码就回车三次即可。
    Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]
  • 如果输出一张RSA密码图，就说明加密成功，密钥文件一般在User-[你的系统用户名]-ssh.，后缀名为pub的文件。
  • 通过记事本或其他方式大概打开文件，复制其中的SSH密钥字符串。
  • 然后在云端设置我们的SSH密钥，这样我们的本地博客就与云端建立联系了。这里我们以Github为例，路径为Public porfile-SSH and GPD keys-New SSH key，黏贴SSH密钥字符串并保存，Title可无视。
